写在前面

Markdown 是一种标记型编辑语法，最终的展示效果因渲染器的版本而定，因此有可能出现一种渲染器能够识别而另一种渲染器不识别。比如 typora 支持的 [TOC] 在 GitHub 上不支持，有道云笔记的"===="字体背景高亮在 typora 不支持等。因此对于不识别的语法我们要着重注意并尽量避开，或者对于不识别的语法进行 HTML 排版以达到想要的目的，md 的输出也以 PDF 或者 HTML 为主最好。

Markdown 的语法很多都不止一种写法，以下示例是我常用的语法规则。

基本语法

  1. 标题语法

    一级标题  #
    二级标题  ##
    三级标题  ###
    四级标题  ####
    五级标题  #####
    六级标题  ######

  2. 引用

    xxxxxxxxxx
    > 引用内容

  3. 分割线

    xxxxxxxxxx
    ---

  4. 删除线

    xxxxxxxxxx
    ~~删除线~~

  5. 字体加粗

    xxxxxxxxxx
    **山海**

  6. 无序列表

    xxxxxxxxxx
    - Java
    - C++

  7. 有序列表

    xxxxxxxxxx
    1. Pyhton
    2. C#

  8. 任务列表

    xxxxxxxxxx
    - [ ] 任务1
    - [ ] 任务2

    列表都可以进行嵌套使用

  9. 插入链接

    xxxxxxxxxx
    [外部链接](https://tallon.ink/)

  10. 插入图片

    xxxxxxxxxx
    ![外部图片](https://mrtallon.gitee.io/img/diary1.jpg)

  11. 插入表格

    xxxxxxxxxx
    |      |      |
    | ---- | ---- |
    |      |      |

  12. 代码块

    xxxxxxxxxx
    ```
    代码内容
    ```

  13. 脚注

    xxxxxxxxxx
    我需引用脚注说明[^foot]
    [^foot]:这里是脚注的具体内容

  14. 目录

    xxxxxxxxxx
    [TOC]

    使用前请确认渲染器是否支持目录语法

初级进阶：HTML

  1. 下划线

    xxxxxxxxxx
    <u>泛舟沧海，立马昆仑</u>

  2. 居中

    xxxxxxxxxx
    <center>心有敬畏，行有所止</center>

  3. 右对齐

    xxxxxxxxxx
    <div style="text-align:right">道德经</div>

  4. 字体颜色和大小

    xxxxxxxxxx
    <font color="#187892" size="number">浮舟沧海</font>

    颜色可选择16进制色号，也可以直接输入“red”等单词颜色，大小可以选择“+2”等与常规进行区分

以上很多语法都可以组合使用，大家可自行尝试
